技术考虑：
1、之所以抽象这一层，目前的一个考虑就是需要在消息队列层面区分玩家，因为在一个时间片内，可能玩家会触发n次操作，那么是否这n个操作请求
region都需要一个个处理，是有待商榷的
2、假设我们为每个玩家先设置一个队列，当时间片消息来到的时候，我们可以对这些消息先做一些过滤，比如：
    a)合并重复的移动操作，只保留最新的一次移动请求(比如玩家频繁的用鼠标请求移动到这里那里)
    b)发现玩家有多次提交的操作，但是当前并非玩家的回合(或者没轮到他的cd时间),那么region无需丢弃该消息，只需要把消息退回玩家的消息队列即可
    c)可以从服务端针对性的控制玩家的输入
